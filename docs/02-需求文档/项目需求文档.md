# Labex 实验教学管理系统 - 需求分析与系统设计文档

## 一、项目概述
本项目为**实验教学管理平台**，面向教师与学生提供实验、考试、成绩全流程管理能力。
- 旧系统：Spring MVC 4 + JSP + jQuery + Bootstrap + Spring JDBC + MySQL + Tomcat 9
- 核心问题：前后端耦合、技术老旧、维护困难、部署复杂、安全性不足
- 重构目标：前后端分离、现代化技术栈、容器化部署、提升开发效率与用户体验

---

## 二、业务需求（完整功能清单）
### 2.1 角色定义
- 教师：管理班级、学生、实验、讲义、成绩、考试、题库
- 学生：学习讲义、完成实验、提交答案、查看成绩、修改个人密码

### 2.2 核心业务模块
#### 1. 用户认证模块
- 用户登录（教师/学生）
- 用户注销
- 密码修改
- 权限控制与身份校验

#### 2. 教师模块
- 班级管理：增删改查、启用/禁用、IP 限制开关
- 学生管理：增删改查、批量导入、启用/禁用、重置密码/重置IP
- 讲义管理：上传文件、列表展示、编辑
- 实验管理：创建/编辑实验、管理实验题目、设置标准答案与分值、启用/禁用
- 习题管理：题目增删改查、分类管理
- 成绩管理：查看学生报告、在线评分、批量评分、成绩导出

#### 3. 学生模块
- 讲义学习：查看与阅读教师发布讲义
- 实验操作：实验列表、题目列表、在线答题、自动/手动保存答案、题目导航
- 个人中心：修改登录密码

#### 4. 考试与统计模块
- 题库管理
- 考试管理
- 成绩统计、数据分析

---

## 三、非功能需求
1. 安全性：密码加密、接口权限控制、防SQL注入、防XSS/CSRF
2. 可用性：7×24小时稳定运行，操作简单直观
3. 可维护性：模块化、低耦合、注释规范
4. 可扩展性：支持功能扩展、水平扩展
5. 部署性：支持容器化一键部署、环境一致性

## 五、数据库设计（核心表结构）
### 5.1 用户相关表
- **t_teacher**（教师表）
  - teacher_id：主键，自增
  - teacher_account：账号，唯一
  - teacher_password：密码（MD5，重构后改为BCrypt）
  - teacher_name：姓名

- **t_student**（学生表）
  - student_id：主键，自增
  - student_no：学号，唯一
  - student_name：姓名
  - student_password：密码
  - clazz_no：班级编号
  - state：状态（0正常/1禁用）
  - ip：绑定IP

- **t_clazz**（班级表）
  - no：班级编号，主键
  - memo：描述
  - state：IP限制状态（0关闭/1开启）

### 5.2 实验相关表
- **t_experiment**（实验表）
  - experiment_id：主键
  - experiment_name：实验名称
  - experiment_content：实验内容
  - state：启用状态

- **t_experiment_item**（实验题目表）
  - experiment_item_id：主键
  - experiment_id：所属实验
  - experiment_item_name：题目名称
  - experiment_item_content：题目内容
  - experiment_item_answer：标准答案
  - experiment_item_score：分值

### 5.3 答题与成绩表
- **t_student_item**（学生答题表）
  - student_item_id：主键
  - student_id：学生ID
  - item_id：题目ID
  - content：答案内容
  - score：得分
  - fill_time：提交时间

- **t_score**（成绩表）
  - score_id：主键
  - student_id：学生ID
  - experiment_id：实验ID
  - score：总分

### 5.4 日志与其他表
- t_sys_log：系统日志
- t_student_log：学生行为日志
- t_lecture：讲义表
- t_question：习题表
- t_sys_config：系统配置表

## 六、核心业务流程
### 6.1 用户登录流程
1. 前端提交账号密码
2. 后端验证身份
3. 生成JWT令牌返回
4. 前端存储Token并在请求头携带
5. 按角色跳转到对应页面

### 6.2 学生实验流程
1. 学生进入实验列表
2. 选择实验查看题目
3. 在线填写答案，自动保存
4. 提交答案
5. 教师评分后可查看成绩

### 6.3 教师管理流程
1. 创建/管理班级
2. 批量导入/管理学生
3. 创建实验与题目
4. 批改学生作业
5. 统计与导出成绩

### 6.4 学生批量导入流程
1. 上传CSV文件
2. 解析学号、姓名
3. 验证学号格式（8位数字）
4. 批量入库，默认密码=学号
5. 返回导入结果

### 7.1 总体架构
- 前端：Vue3单页应用，独立部署
- 网关/API层：RESTful接口
- 后端：Spring Boot微服务/单体应用
- 数据层：MySQL + Redis
- 部署：Docker容器化

### 7.2 分层架构
- 表现层：Controller
- 业务层：Service
- 数据访问层：Mapper/MyBatis-Plus
- 实体层：Model/DTO/VO
