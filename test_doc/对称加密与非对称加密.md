# 对称加密与非对称加密

## 对称加密

### 原理
加密和解密使用相同的密钥。

### 常用算法

| 算法 | 密钥长度 | 安全性 |
|------|----------|--------|
| AES | 128/192/256位 | 高 |
| DES | 56位 | 低（已破解） |
| 3DES | 168位 | 中 |

### 优缺点
**优点**：速度快、资源消耗低
**缺点**：密钥分发困难

## 非对称加密

### 原理
使用公钥和私钥配对进行加密解密。

### 常用算法

| 算法 | 应用 |
|------|------|
| RSA | 密钥交换、数字签名 |
| ECC | 移动端加密 |
| DH | 密钥交换 |

### 优缺点
**优点**：密钥分发方便、支持数字签名
**缺点**：速度慢

## 混合加密

1. 使用非对称加密传输对称密钥
2. 使用对称加密加密实际数据'
